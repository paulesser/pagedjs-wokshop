# HS Mainz pagedjs workshop

## setup

Today we use the preconfigured pagedjs polyfill. You can also configure pagedjs yourself using the paged.js or the npm package.
[paged documentation](https://pagedjs.org/documentation/2-getting-started-with-paged.js/#using-paged.js-as-a-polyfill-in-web-browsers)

```HTML
<script src="./paged.polyfill.js" defer></script>
```

The polyfill runs automatically on load. to prevent that we can add `PagedConfig` with `{ auto: false }` to the [global window object](https://developer.mozilla.org/en-US/docs/Glossary/Global_object)

```HTML
<script> window.PagedConfig = { auto: false };</script>
```

Now we add a button that starts pagedjs to our body:

```HTML
<button onclick="window.PagedPolyfill.preview()" class="no-print"> preview</button>
```

We also need to add the `interface.css` from the [official gitlab](https://gitlab.coko.foundation/pagedjs/interface-polyfill) to see the paginated pages

```HTML
<link rel="stylesheet" href="./interface.css" />
```

Your HTML should look something like this:

```HTML
<!DOCTYPE html>
<html lang="en">
    <head>
        <script>
            window.PagedConfig = { auto: false };
        </script>
        <script src="./paged.polyfill.js" defer></script>
        <link rel="stylesheet" href="./interface.css" />
    </head>
    <body>
        <button onclick="window.PagedPolyfill.preview()" class="no-print"> preview</button>
    </body>
</html>
```

now you can add your css file, content and elements and start designing.
